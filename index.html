Core Frameworks

This project analyzes three fundamental models of Human-Computer Interaction as described in Dix et al., Human–Computer Interaction (Ch. 12).

Norman's Stages of Action
A descriptive model focusing on user psychology. It identifies the Gulf of Execution (difficulty using the device) and Gulf of Evaluation (difficulty understanding feedback). This helps explain why errors occur in safety-critical systems.
GOMS Analysis
A predictive model describing user procedural knowledge. GOMS stands for Goals, Operators, Methods, and Selection Rules. It models expert users performing routine tasks efficiently.
KLM Predictions
The Keystroke-Level Model predicts execution time using primitive operators like K (Keystroke), P (Pointing), and H (Homing). It assumes error-free performance and compares interface speeds.

Why These Models Matter

For medical devices like syringe pumps, GOMS and KLM predict expert speed but assume perfect performance. Norman's model reveals where errors occur during interpretation and evaluation. Combining all three models provides complete safety analysis.
