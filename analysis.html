Task: Setting a 10ml/hr Infusion Rate

Analyzing the interaction steps required for a nurse to set a dosage on a standard medical pump. This comparison highlights where GOMS predicts expert speed and where Norman's Action Cycle identifies potential for life-threatening errors.

Norman's 7 Stages vs. GOMS Framework
Interaction Stage Norman's Action Cycle GOMS Component Safety Risk
Initial State Goal: Intention to administer 10ml/hr. Goal: SET-INFUSION-RATE. Errors here are high-level clinical mistakes.
Planning Plan: Decide to use "Rate" button and numeric keys. Method: Keypad-Entry Method. Gulf of Execution: Can the nurse find the "Rate" button?
Motor Action Execute: Pressing [Rate], [1], [0], [Enter]. Operators: Press-Key(Rate), Press-Key(1), Press-Key(0). GOMS predicts exact physical time using KLM.
Perceiving Perceive: Seeing the 10ml/hr confirmation on LCD. (Not Modeled in GOMS) Gulf of Evaluation: If screen is dim, nurse may miss error.
Evaluation Evaluate: Does screen match doctor's order? Selection Rule: If correct, start pump. Critical final check to prevent overdose.

Critical Analysis Summary

As discussed in Dix et al., Chapter 12, hierarchical models like GOMS are ideal for calculating the efficiency of repetitive tasks for expert users. However, medical devices must be evaluated using Norman's Cycle because the most dangerous errors occur during Interpretation and Evaluation — stages GOMS assumes will happen perfectly.
